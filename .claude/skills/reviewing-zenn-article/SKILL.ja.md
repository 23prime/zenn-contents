# Zenn 記事のレビュー

## 概要

Zenn 記事の品質・正確性・文体をレビューする。すべてのチェックを実行し、結果を報告する。自動修正できる問題は修正し、それ以外は手動対応のために報告する。

## ワークフロー

### 1. 記事の確認

`articles/<slug>.md` の記事ファイルを読み込む。

### 2. 自動チェックの実行

以下を順番に実行する（先に修正、次にチェック）：

```bash
mise run fix
mise run check
```

`mise run fix` 実行後も残っているエラーや警告を報告する。

### 3. コンテンツ品質レビュー

以下の観点で記事をチェックする。結果は構造化されたリストで報告する。

#### 構成

- [ ] 背景と記事の対象範囲を説明するはじめに（導入）がある
- [ ] 本編セクションが論理的な順序で並んでいる
- [ ] 要点を振り返るまとめ・結論がある
- [ ] h2〜h4 のみ使用している（本文中に h1 がない）
- [ ] 各セクション見出し（h2〜h4）の先頭に内容にフィットする絵文字がある

#### ターゲット読者

- [ ] 対象読者が明確（明示されているか、一貫して暗示されている）
- [ ] 前提知識が対象読者のレベルに合っている
- [ ] 対象読者が理解できない専門用語が説明なく使われていない

#### コードサンプル

- [ ] コードサンプルが正確で実行可能
- [ ] 各コードサンプルが周囲のテキストで説明されている
- [ ] 有用な場合はコードブロックにファイル名が指定されている（例: ` ```toml:mise.toml `）

#### 文体・表現

- [ ] 文章が明確で簡潔
- [ ] 不自然または読みにくい日本語表現がない
- [ ] 全体を通してトーンと文体が統一されている
- [ ] 段落が空行で区切られている（単一改行ではない）
- [ ] `docs/WRITING_RULES.md` に定義された全ルールに従っている（ファイルを読んで各ルールを確認する）

#### 参考リンク

- [ ] 本文中にインラインで引用した URL（例：`（[参考](https://...)）`）がすべて末尾の `## 参考` セクションにも掲載されている

#### Zenn コミュニティガイドライン

`references/zenn-guideline.md` を読んで以下を確認する：

- [ ] 実体験や独自の考察が含まれている（公式情報の引用だけでない）
- [ ] タイトルが内容を正確に反映している（誇張やクリックベイトでない）
- [ ] 冒頭に概要があり、対象読者が明示されている
- [ ] 広告・採用宣伝を主目的にしていない
- [ ] 引用時に出典が明記されており、必要な範囲にとどまっている

#### Zenn ルール準拠

- [ ] `published` フィールドが正しく設定されている（下書きは `false`、公開は `true`）
- [ ] `emoji` が絵文字1文字
- [ ] `topics` が5個以下
- [ ] HTML タグが使われていない（`<br>` を除く）
- [ ] 画像パスが絶対パス（`/images/<slug>/...`）で、相対パスでない

### 4. 誤字チェック

記事全文を読み、以下を確認する：

- スペルミス（日本語・英語）
- 誤字（例: 誤った漢字の使用）
- 明らかな誤変換

### 5. 結果の報告

すべての指摘事項をまとめて報告する：

```md
## レビュー結果

### 自動チェック
<合格 / エラー一覧>

### コンテンツ品質
<チェックリスト結果 — 問題のある項目のみ記載>

### 誤字
<場所と修正案を含む誤字一覧、または「なし」>

### まとめ
<総合評価と推奨される対応>
```

問題がない場合はその旨を明示する。
